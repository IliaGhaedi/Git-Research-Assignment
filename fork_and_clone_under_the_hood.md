مراحل fork:
یه ریپو جدید دقیقا مشابه ریپو اصلی (شامل branch ها، commit ها، tag ها و ...) رو remote جدید با نام origin ایجاد میکنه
ارتباط این ریپو با ریپو اصلی علامت زده میشه تا بعدا با استفاده از pull توانایی اعمال تغییرات رو در ریپو جدید رو دارا باشه

مراحل clone:
گیت با سرور ارتباط برقرار میکنه و تمام object ها و ref ها رو دانلود میکنه
یه فولدر لوکال میسازه و در git. اطلاعات رو ذخیره میکنه ( شامل git/objects. و git/refs. و git/config. )
بعد از clone شدن، شاخه master, checkout میشه و فایل ها ظاهر میشن
